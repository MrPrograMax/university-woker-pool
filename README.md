# Mini worker pool (Go)

## Задание

Необходимо на C# либо на вашем языке программирования если тот поддерживает абстракцию "некий пул потоков" коим в C# является SyncronizationContext реализовать по сути свой небольшой, элементарный пул потоков на 4 потока, который через некий интерфейс принимает на исполнение методы, а те выполняются на созданных потоках. Вначале своей жизни созданный поток должен вывести в консоль номер потока. Запланированные методы должны вывести номер текущего потока, на которм они работают.

## Запуск

```bash
go run .
```

## Пример вывода

```
Started new worker: #1
Started new worker: #2
Started new worker: #3
Started new worker: #4
Press Enter to enqueue a few more jobs...
2
2
4
1
...
Send executed on worker #3
Press Enter to stop...
Finished
```

## Нюансы (важно для Go)

* В Go нет стабильного `ThreadId` как в C#: goroutine могут выполняться на разных OS-потоках.
* Поэтому “номер потока” в задании здесь — это **ID воркера пула (1..4)**.
* Все задачи (включая вложенные `Post` внутри задачи) выполняются одним из этих 4 воркеров.
